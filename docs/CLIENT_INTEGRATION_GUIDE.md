# Client Integration Guide

## Predict Age From Client Audio

Call `POST /api/predict-age` with `multipart/form-data`.

```bash
curl -X POST https://<client-domain>/api/predict-age \
  -F "file=@caller.wav"
```

Successful response:

```json
{
  "predicted_age_group": "Adult",
  "confidence": 0.91,
  "confidence_level": "high",
  "class_probabilities": {
    "Adult": 0.91,
    "Middle_Age": 0.04,
    "Senior": 0.03,
    "Teen": 0.02
  },
  "model_version": "wav2vec_75k",
  "processing_time_ms": 184
}
```

## Connect a Twilio Phone Number

Configure the Twilio phone number voice webhook:

- Recording flow: `POST https://<client-domain>/api/twilio/voice`
- Realtime assistant flow: `POST https://<client-domain>/api/twilio/realtime-voice`

Set `PUBLIC_BASE_URL=https://<client-domain>` so Twilio can connect to the WebSocket URL generated by the API.

When `TWILIO_AUTH_TOKEN` is set, incoming Twilio webhooks are signature validated.

## Reports

Realtime calls write artifacts to:

```text
data/realtime_conversations/<call_sid>/
```

Post-call reports are generated in:

```text
data/realtime_conversations/<call_sid>/reports/
```

The main client-facing report is `combined_call_report.json`. It includes VoiceAge result, conversation summary, latency metrics, system checks, and demo readiness recommendation.

Recording-only predictions write JSON logs to:

```text
data/twilio_predictions/
```

## Dashboard Integration Suggestions

Use `GET /health` for service status, model version, model path, and active device.

For prediction history, ingest `data/twilio_predictions/*.json` and `data/realtime_conversations/*/reports/combined_call_report.json`.

Recommended dashboard fields:

- Predicted age group and confidence
- Confidence level
- Model version
- Caller-only audio duration
- Assistant audio excluded status
- Realtime transcript availability
- Total response latency
- Report recommendation
